# 2026-03-10 工时统计与分析系统设计方案

## 1. 目标
构建一个能够自动读取 Excel 工时表、持继滚动统计、并提供多维度动态图表展示的网页版程序。

## 2. 核心架构 (Hybrid Architecture)
- **后端 (Backend)**: Python (FastAPI) + SQLite 数据库
    - 负责 Excel 解析、数据持久化（支持滚动更新）、多维度计算。
    - 数据库文件存储在 D 盘，确保历史数据不丢失。
- **前端 (Frontend)**: React + Vite + ECharts
    - 负责精美的 UI 展示、交互式图表、多级筛选过滤。
- **环境隔离**: 
    - 所有依赖项和虚拟环境将安装在 `D:\Antigravity\Project-timesheet\.venv`。

## 3. 功能模块
### 3.1 数据管理
- **Excel 导入**: 支持拖拽上传，自动解析员工、项目、工时等字段。
- **增量更新**: 自动识别新上传的数据，避免重复导入，支持历史数据的追加和修正。

### 3.2 统计维度
- **时间维度**: 周、月、季度、半年、年度。
- **层级维度**: 部门 -> 团队 -> 员工。
- **项目维度**: 项目对比、项目与非项目（假勤）占比。

### 3.3 可视化看板 (Visualization)
- **柱状图**: 部门/项目工时对比。
- **折线图**: 工时随时间的变化趋势。
- **饼图**: 工时构成占比（项目 vs 非项目）。
- **行军图 (Gantt/March)**: 项目进度与工时投入的时间线展示。
- **桑基图 (Sankey)**: 工时在部门与项目间的流动分布。
- **热力图 (Heatmap)**: 员工忙闲程度的日历视图。

## 4. 交互设计
- **动态筛选器**: 
    - 日期范围选择器。
    - 项目/部门多选下拉菜单。
- **联动效应**: 更改筛选条件，所有图表同步刷新。
- **交叉对比**: 支持选取两个项目或两个部门进行并列对比。

## 5. 部署与迁移
- **当前阶段**: 在用户 D 盘运行，提供“一键启动”脚本。
- **迁移阶段**: 整个文件夹拷贝至公司服务器，配置 Python 环境即可运行。
